# Data description

This directory contains the answer keys and specialist ratings used in the
qualitative evaluation reported in the paper.

## `answer_keys.csv`

The purpose of this file is to identify the reference words used in the
qualitative evaluation and document the structural profile assigned to each
word. It contains one row for each of the 35 answer keys.

| Column | Description |
| --- | --- |
| `answer_key_id` | Identifier assigned to the answer key in this dataset. |
| `answer_key_profile` | Structural profile associated with the answer key. |
| `profile_description` | Explanation of the syllable-count and syllabic-pattern criteria represented by the profile. |
| `answer_key` | Reference word used to generate distractor candidates. |

The possible values of `answer_key_profile` are:

| Value | Description |
| --- | --- |
| `canonical_disyllabic` | Canonical answer key with one or two syllables. |
| `canonical_trisyllabic` | Canonical answer key with three syllables. |
| `canonical_polysyllabic` | Canonical answer key with four or more syllables. |
| `noncanonical_disyllabic` | Non-canonical answer key with one or two syllables. |
| `noncanonical_polysyllabic` | Non-canonical answer key with four or more syllables. |

## `qualitative_evaluations.csv`

The purpose of this file is to present the specialist evaluation of every
candidate generated by the LLM-based and rule-based approaches. Each row
identifies the answer key, generation method, structural candidate group,
ranking position, candidate word, and assigned rating. The file contains 5,600
candidate evaluations.

| Column | Description |
| --- | --- |
| `answer_key_id` | Links the candidate evaluation to `answer_keys.csv`. |
| `answer_key_profile` | Structural profile associated with the answer key. |
| `method` | Candidate-generation method: `llm` or `rule_based`. |
| `candidate_pattern` | Syllabic pattern required for the candidate: `canonical` or `noncanonical`. |
| `syllable_count_relation` | Whether the candidate has the `same` or a `different` number of syllables relative to the answer key. |
| `rank` | Candidate position within its generated list, from 1 to 20. |
| `candidate` | Evaluated distractor candidate. |
| `rating` | Specialist rating assigned to the candidate. |

The possible ratings are:

| `rating` | Interpretation |
| --- | --- |
| `ideal` | Candidate considered particularly suitable as a distractor. |
| `usable` | Candidate considered usable as a distractor. |
| `poor` | Candidate considered unsuitable as a distractor. |
| `not_a_noun` | Candidate considered invalid because it is not an appropriate noun. |

## Structural candidate groups

The combination of `candidate_pattern` and `syllable_count_relation` identifies
the four candidate lists generated for every answer key and method:

| Candidate pattern | Syllable-count relation |
| --- | --- |
| `canonical` | `same` |
| `canonical` | `different` |
| `noncanonical` | `same` |
| `noncanonical` | `different` |
